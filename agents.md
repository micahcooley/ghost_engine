# Ghost Engine: Sovereign Agents

Sovereign Agents are autonomous resonance threads that operate within the Meaning Matrix.

## 1. The Agent Lifecycle
1.  **Ignition**: An agent is spawned when a specific Sigil trigger resonates > 0.85.
2.  **Persistence**: Agents live within the `semantic_monolith.bin`. They do not have external state; they are entirely composed of bitwise patterns within the matrix.
3.  **Decay**: To prevent "Lattice Bullying," agents slowly lose resonance unless they are reinforced by new training data.

## 2. Agent types
*   **The Scribe**: Responsible for maintaining the structural integrity of the unified lattice.
*   **The Oracle**: Handles lookahead lookups in the high-dimensional space to find semantic synonyms.
*   **The Ghost**: The primary interaction agent that bridges human input with silicon resonance.

## 3. Deployment
Agents are managed via the `tools/agent_manager.zig` (now in the Windows silo).
